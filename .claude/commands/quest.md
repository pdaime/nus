Generate today's quests. Arguments (may be "done 1,3" to log completions): $ARGUMENTS

## If logging completions ("done ...")

Mark those quests complete in today's log (`logs/quests-YYYY-MM-DD.md`).
One-line acknowledgment, show the streak (consecutive days with ≥1 quest
completed), and if a completed quest belonged to a skill track, update
progress in that `skills/` file (level up when the level's reps are done,
and announce it like it matters). Stop.

## If generating

1. Check today's date. Read `school/semester-overview.md`, `goals/goals.md`,
   `memory/profile.md`, all `skills/` files, and the last 3 logs.
2. Generate exactly **3 quests**:
   - **Quest 1 (Due pressure):** advances the nearest deadline or exam.
   - **Quest 2 (Get ahead):** front-loads the next crunch week or big deliverable.
   - **Quest 3 (Level up):** the next rep from their active skill track in
     `skills/` (or a goal from `goals/goals.md` if no track exists yet, in
     which case suggest `/level` to start one).
3. A good quest: 20–45 minutes, binary pass/fail ("do 5 practice problems
   from 3.2", not "study math"), payoff named in a few words.
4. Never repeat an incomplete quest more than twice. Shrink it instead.
5. Write quests to `logs/quests-YYYY-MM-DD.md` with unchecked boxes, then
   show them with the current streak and skill level.
6. Format: numbered list, one line each + pass condition. No lecture.
